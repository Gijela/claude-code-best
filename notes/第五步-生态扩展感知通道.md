# 第五步：生态扩展感知通道

**回答问题**：生态扩展背后有一套让模型"知道自己能干什么"的感知通道，具体是什么？

---

## 问题：模型怎么知道自己能干什么？

模型收到了工具列表（name + description + inputSchema），但：
- 工具太多（50+），描述占用大量 token
- MCP 工具动态变化，模型不知道有哪些 server
- 技能（/斜杠命令）需要发现机制

解决：**多层感知通道**。

---

## 第一层：Tool.prompt() —— 工具使用指南

每个工具通过 `prompt()` 方法返回详细使用说明，注入到 System Prompt：

```typescript
// 工具不只声明"我能干什么"，还告诉模型"怎么用我"
prompt(): string | null
```

例如：
- **BashTool.prompt()**：告诉模型命令执行规则、超时机制、后台化策略
- **FileEditTool.prompt()**：告诉模型必须先 Read、引号处理规则

这是第一层感知：**工具自己告诉模型怎么用它**。

**与 Prompt 编排联动**：Tool.prompt() 返回的内容会被组装到 System Prompt 的「工具使用指南」区块，是动态组装的重要组成部分。

---

## 第二层：ToolSearch —— 工具发现

当可用工具超过 50 个时，模型可能不知道该用哪个。ToolSearch 提供了**工具发现机制**：

```
输入: query = "database connection"
     ↓
1. 精确匹配: 检查是否有工具名完全匹配
2. MCP 前缀匹配: "mcp__postgres" → 匹配所有 postgres 相关工具
3. 关键词拆分: ["database", "connection"]
4. 加权评分:
   - 工具名精确匹配: 10 分
   - 工具名部分匹配: 5 分
   - searchHint 匹配: 4 分
   - 描述匹配: 2 分
5. 按分数排序，返回 top-N
```

这是第二层感知：**让模型在众多工具中找到目标**。

**与工具治理联动**：ToolSearch 的搜索范围来自工具治理流水线的分层组装结果，MCP 工具通过 `mcpInfo` 字段参与匹配。

---

## 第三层：技能系统 —— /斜杠命令

技能是预定义的任务模板，通过 `/斜杠命令` 触发：

```
/commit  →  执行 git commit 流程
/review  →  代码审查
/test    →  运行测试
```

技能通过 `SkillTool` 暴露给模型。`alwaysLoad: true` 确保技能工具在 turn 1 就可见。

这是第三层感知：**预定义的任务模板，让模型知道"能做这些事"**。

**与 Agent 调度联动**：技能本质上是一组预编排的工具调用序列。调度系统执行 SkillTool 时，会解析技能定义并按序执行其中的工具调用。

---

## 第四层：MCP 工具发现

MCP Server 提供的工具通过 `mcpInfo` 字段标记来源：

```
mcp__<server_name>__<tool_name>
```

**发现机制**：
1. MCP Server 连接时，发送 `tools/list` 请求获取工具列表
2. 工具通过 `mcpInfo` 字段标记来源
3. 模型通过 ToolSearch 或直接调用发现这些工具

这是第四层感知：**外部服务的工具自动注入，让模型知道"还能连接这些系统"**。

**三系统协同**：
- **工具治理**：MCP 工具通过 `mcpInfo` 参与 ToolSearch 匹配；受 blanket deny 规则约束
- **Prompt 编排**：MCP 工具的 name/description/inputSchema 注入到工具列表区块
- **Agent 调度**：调度系统决定 MCP 工具调用的并发策略（取决于 `isConcurrencySafe()`）

---

## 四层感知通道与三系统的协同关系

```
┌─────────────────────────────────────────────────────────────┐
│                    Prompt 编排机制                           │
│        Tool.prompt() → System Prompt 动态区                  │
│        MCP 工具描述 → 工具列表区块                            │
│                          ↓                                  │
├─────────────────────────────────────────────────────────────┤
│                    Agent 调度系统                            │
│        技能 → 预编排工具序列 → 调度执行                        │
│        MCP 工具 → 并发判断 → 流式/批量执行                     │
│                          ↓                                  │
├─────────────────────────────────────────────────────────────┤
│                    工具治理流水线                            │
│        ToolSearch 搜索范围 ← 分层组装结果                     │
│        MCP 工具 ← blanket deny 规则                          │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                 生态扩展感知通道                             │
│   Tool.prompt() → ToolSearch → 技能 → MCP 发现               │
│   "让模型知道自己能干什么"                                    │
└─────────────────────────────────────────────────────────────┘
```

## 与简单 Agent 的差距

| 维度 | 简单 ReAct | Claude Code |
|------|-----------|-------------|
| **工具发现** | 只有 name/description | ToolSearch 加权搜索 |
| **使用指南** | 无 | Tool.prompt() 详细说明 |
| **任务模板** | 无 | 技能系统预定义 |
| **外部扩展** | 无 | MCP 自动发现 + 权限控制 |
